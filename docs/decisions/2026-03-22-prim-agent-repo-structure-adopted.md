# Agent-optimised repo structure adopted

**Date:** 2026-03-22
**Project:** {{JIRA_PROJECT_KEY}} ({{CLIENT_NAME}}) — pilot project
**Source:** Design session

## Decision

{{AGENCY_NAME}} will use an agent-optimised GIT project repo structure for all client projects,
piloted first on {{CLIENT_NAME}} ({{JIRA_PROJECT_KEY}}). The project repo becomes the single source of truth
for all project context, replacing scattered storage across JIRA, Slack, Google Docs, and
individual Claude chat sessions.

## Structure adopted

- `.agent/` — agent front door (CONTEXT.md auto-generated, skills, instructions)
- `docs/` — permanent knowledge (decisions, specs, client-comms)
- `tasks/` — sprint-lived context (current-sprint.md auto-synced from JIRA)
- `memory/` — session-lived context (quick-tasks.md, session notes)
- `digest/` — n8n PR landing zone (daily Slack+Gmail digest)
- `scripts/` — automation (generate-context.py, route-digest.py)
- `.github/workflows/` — context-update.yml, digest-router.yml

## Key design decisions

- CONTEXT.md is always auto-generated by GitHub Actions — never human-written
- CONTEXT.md is capped at ≤300 words so it fits any context window and is fast to paste
- Website repo added as git submodule at `website/`
- Shell scripts replaced by agent skills (`.agent/skills/*.md`)
- SoW split into shared (this repo) and private (Google Drive) versions
- Digest format: semicolon-delimited CSV with maxsplit=3 parser — content field is safe from delimiter collision

## Automation

- n8n daily-digest: 3 AM → reads Slack + Gmail → Claude classifies → CSV PR → PM reviews
- n8n sprint-sync: Sunday 20:00 → JIRA → tasks/current-sprint.md
- n8n session-webhook: Slack /session → memory/sessions/
- GitHub Actions: regenerates CONTEXT.md on every merge
